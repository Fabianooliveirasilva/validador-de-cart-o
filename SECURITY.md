# Security Policy

Obrigado por se preocupar com a segurança do projeto.

Se você encontrou uma vulnerabilidade, por favor não a publique publicamente. Prefira uma das seguintes opções:

- Utilize o recurso "Security Advisories" do GitHub para reportar a vulnerabilidade de forma privada.
- Ou envie um e-mail para `security@placeholder.example` (substitua pelo contato apropriado do mantenedor) com:
  - Descrição do problema
  - Passos para reproduzir
  - Impacto esperado
  - Versões afetadas

> Nota: `security@placeholder.example` é um placeholder — por favor substitua este e-mail pelo contato real do mantenedor antes de publicar o repositório.

Nosso processo de resposta:

1. Agradecemos o reporte e confirmamos o recebimento.
2. Investigamos e, se necessário, preparamos uma correção priorizada.
3. Trabalhamos com o autor do reporte para validar a correção e coordenar divulgação responsável.

Se você não tiver certeza sobre o canal, abra uma issue pública marcada como `security` e a convertiremos para um relatório privado quando necessário.
